Concept

Everything that can be invented has been invented

Code reusability should be higher. Even highly qualified IT consultants spend days programming the same logical interface to a utility class and extension methods. Stop it. Use open source or ask for an API built to be readily reused.

- Why write it yourself?

Software projects are usually under-estimated. As Jeff Atwood quoted: "The only new software development projects undertaken are those that haven't been done before or those whose predecessors are not publicly available".

Can implement it in a day? Unit testing and full coverage? Maybe two days. Documentation? You're kidding. It'll be a small project just for you. Until the next project.

  • Testing vigourously: unit tests, high code coverage, continiously maintained
  • Documentation: abstracted overall intro and level based documentation, competing with MSDN and PHP docs
  • Extendable: the future is here soon, better make the code extendable to a rational extent

- Coding it yourself vs learning a library

Download the result from github and nuget. Learning to use a well-established library is easier than people think.
